Příkazy pro ovládání Gitu

Základní konfigurace

Pokud chceme klonovat nějaký repozitář z GitHubu, tak použijeme příkaz

git clone https://github.io/nazevrepozitare/neco.git

Na některých pracovištích (školách, firmách, institucích, ...) "běží", tzv. proxy server, který zajišťuje a stará se o komunikaci mezi firemním, školním intranetem a globálním internetem. Právě proto bychom měli provést operaci, která zajistí komunikaci mezi proxy a internetem "otevřeně". Před vytvořením repozitáře provedeme tyto 2 příkazy:

git config --global http.proxy http://adresaproxy:800

git config --global http.sslverify "false"

Pro vytvoření lokálního repozitáře je potřeba využít příkaz:

git init

Když potřebujeme zjistit stav našeho repozitáře, měli bychom použít příkaz:

git status

Nesmíme také zapomenout na nastavení uživatelského účtu pro použití v lokálním repozitáři - to se hodí právě pro ty sítě oddělené proxy:

git --local user.name "nick"

A pro nastavení e-mailové adresy:

git config --local user.email "vasemail@domena.cz"

Pokud se bavíme o nastavení v celém počítači (ve vaší domácnosti, u prarodičů, ...), tak použijeme tyto příkazy, které jsou velice obdobné, až na dvě slůvka:

git config --global user.name "nick"

git config --global user.email "vasemail@domena.cz

Pro zobrazení konfiguračních souborů "na obrazovku" využijeme příkaz:

git --list

Jak se pracuje s lokálním repozitářem?

Budeme určitě muset přesunout všechny soubory z našeho lokálního repozitáře do části zvané staging area:

git add -all nebo git add .

Můžeme samozřejmě do staging area přesouvat soubory samostatně, uveďme si příklad vložení souboru README.md:

git add README.md

Nebo můžeme přesunout všechny textové soubory s libovolným počtem znaků:

git add *.txt

Pokud chceme naopak uvolnit ze staging area všechny textové soubory s libovolným počtem znaků, použijeme:

git reset *.txt

A v případě, že chceme uvolnit ze staging area úplně všechny soubory, tak stačí použít pouze příkaz:

git reset

Spojení lokálního a vzdáleného repozitáře

Pakliže se potřebujeme připojit ke vzdálenému repozitáři ke spojení, je potřeba využít příkaz vyznačený níže, kde origin je obvyklé pojmenování:

git remote add origin https://github.io/nazevrepozitare/neco.git

Ještě bychom mohli zmínit, jak zapíšeme změny do repozitáře s komentářem:

git commit -m "prvni"

Když potřebujeme aktualizovat stav našeho vzdáleného repozitáře (viz. origin), tak jistě využijeme příkaz:

git push -u origin master

Jako podobný se nám může jevit další příkaz, která jen opravuje (fixuje) chyby při aktualizaci

git push -f origin master

Když chceme aktualizovat stav našeho lokálního repozitáře podle toho vzdáleného, použijeme tento příkaz:

git pull origin master

Když chceme vypsat na obrazovku výpis všech aktuálních commitů:

git log
